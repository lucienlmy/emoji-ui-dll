# 性能优化建议

[← 返回主文档](../README.md)

## 概述

本文档提供使用 Emoji Window DLL 时的性能优化建议，帮助你构建高性能的易语言应用。

## 表格控件优化

### 使用虚拟模式处理大数据

当表格行数超过 1000 行时，强烈建议启用虚拟模式：

```
' 创建时启用虚拟模式
表格句柄 = 创建表格 (窗口句柄, 0, 0, 800, 600, 真, 真, #COLOR_BG_WHITE)

' 设置总行数
表格_设置虚拟行数 (表格句柄, 100000)

' 设置数据请求回调（仅加载可见行）
表格_设置虚拟数据回调 (表格句柄, &虚拟数据回调)
```

**性能对比：**
- 普通模式：10000 行约占用 50MB 内存，加载时间 2-3 秒
- 虚拟模式：100000 行仅占用 5MB 内存，加载时间 < 100ms

详见 [DataGridView 文档](controls/datagridview.md#虚拟模式)

## 布局管理器优化

### 批量更新布局

批量修改控件属性后再调用 `UpdateLayout()`，避免频繁触发重绘：

```
' ❌ 错误：每次添加都触发布局更新
.变量循环首 (1, 100, 1, i)
    添加控件到布局 (窗口句柄, 按钮[i])
    更新布局 (窗口句柄)  ' 触发 100 次重绘！
.变量循环尾 ()

' ✅ 正确：批量添加后统一更新
.变量循环首 (1, 100, 1, i)
    添加控件到布局 (窗口句柄, 按钮[i])
.变量循环尾 ()
更新布局 (窗口句柄)  ' 仅触发 1 次重绘
```

### 选择合适的布局类型

不同布局类型的性能特点：

| 布局类型 | 性能 | 适用场景 |
|---------|------|---------|
| 网格布局 | ⭐⭐⭐⭐⭐ | 固定行列，控件数量较多 |
| 停靠布局 | ⭐⭐⭐⭐ | 固定停靠位置，控件数量较少 |
| 流式布局 | ⭐⭐⭐ | 动态换行，控件数量中等 |

## 图片加载优化

### 使用合适的缩放模式

```
' 大图片建议使用等比缩放，减少内存占用
图片框句柄 = 创建图片框 (窗口句柄, 50, 80, 300, 200, 2, #FFF5F7FA)
'                                                        ↑
'                                                  SCALE_FIT (2)
```

**缩放模式性能对比：**
- `SCALE_NONE (0)`: 最快，但大图片可能超出显示区域
- `SCALE_FIT (2)`: 推荐，性能和效果平衡
- `SCALE_STRETCH (1)`: 较慢，可能导致图片变形

### 图片内存管理

从内存加载图片时，使用程序集变量避免内存泄漏：

```
.程序集变量 全局_图片数据, 字节集  ' ✅ 使用程序集变量

' ❌ 错误：局部变量会被释放
.子程序 加载图片
.局部变量 图片数据, 字节集  ' 子程序返回后被释放！
图片数据 = HTTP读文件 ("...")
从内存加载图片 (图片框句柄, 取变量数据地址 (图片数据), ...)
```

详见 [FAQ - 图片加载问题](faq.md#图片加载问题)

## 进度条优化

### 不确定模式的使用

不确定模式使用定时器驱动动画，不会阻塞主线程：

```
' 适用于无法确定进度的场景
设置进度条不确定模式 (进度条句柄, 真)

' 任务完成后恢复确定模式
设置进度条不确定模式 (进度条句柄, 假)
设置进度条值 (进度条句柄, 100)
```

## 事件回调优化

### 避免在回调中执行耗时操作

回调函数在主线程中执行，耗时操作会阻塞 UI：

```
' ❌ 错误：在回调中执行耗时操作
.子程序 按钮点击回调, , , stdcall
.参数 按钮ID, 整数型

' 这会阻塞 UI 5 秒！
延时 (5000)
信息框 ("完成", 0, "")


' ✅ 正确：使用线程处理耗时操作
.子程序 按钮点击回调, , , stdcall
.参数 按钮ID, 整数型

启动线程 (&耗时任务线程, , )


.子程序 耗时任务线程

' 在线程中执行耗时操作
延时 (5000)

' 完成后更新 UI
信息框 ("完成", 0, "")
```

## 主题切换优化

### 减少主题切换频率

主题切换会刷新所有控件，频繁切换可能影响性能：

```
' ❌ 避免：频繁切换主题
.变量循环首 (1, 100, 1, i)
    设置暗色模式 (真)
    延时 (100)
    设置暗色模式 (假)
.变量循环尾 ()

' ✅ 推荐：仅在必要时切换
.如果真 (用户点击暗色模式按钮)
    设置暗色模式 (真)
.如果真结束
```

## 控件数量优化

### 控件数量建议

| 控件类型 | 推荐数量 | 最大数量 |
|---------|---------|---------|
| 按钮 | < 50 | 200 |
| 编辑框 | < 20 | 100 |
| 列表框项 | < 1000 | 10000 |
| 表格行（普通模式） | < 1000 | 5000 |
| 表格行（虚拟模式） | 无限制 | 1000000+ |

### 动态创建和销毁

对于临时控件，使用完后及时销毁：

```
' 创建临时对话框
对话框句柄 = 创建Emoji窗口 ("临时对话框", 400, 300)

' 使用完后销毁
destroy_window (对话框句柄)
```

## 内存使用监控

### 典型内存占用

| 场景 | 内存占用 |
|------|---------|
| 空窗口 | ~2MB |
| 10 个按钮 | ~3MB |
| 100 个按钮 | ~8MB |
| 1000 行表格（普通模式） | ~15MB |
| 100000 行表格（虚拟模式） | ~5MB |
| 加载 1MB 图片 | ~4MB |

## 性能测试工具

### 易语言性能测试示例

```
.版本 2

.子程序 性能测试

.局部变量 开始时间, 长整数型
.局部变量 结束时间, 长整数型
.局部变量 耗时, 长整数型

开始时间 = 取启动时间 ()

' 执行要测试的操作
.变量循环首 (1, 1000, 1, i)
    添加列表项_文本 (列表框句柄, "项目" + 到文本 (i))
.变量循环尾 ()

结束时间 = 取启动时间 ()
耗时 = 结束时间 - 开始时间

调试输出 ("耗时：", 耗时, "毫秒")
```

## 性能优化检查清单

- [ ] 表格超过 1000 行时使用虚拟模式

- [ ] 图片使用 `SCALE_FIT` 缩放模式
- [ ] 从内存加载图片使用程序集变量

- [ ] 控件数量控制在推荐范围内
- [ ] 及时销毁不再使用的控件
- [ ] 避免频繁切换主题
