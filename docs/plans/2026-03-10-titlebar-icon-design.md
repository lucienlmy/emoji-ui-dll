# 标题栏图标修复设计

## 背景

当前自绘窗口在调用 `设置窗口图标()` 后，任务栏可以显示图标，但标题栏左上角不显示图标。

根因不是窗口图标未设置成功，而是窗口标题栏已改为自绘：系统标题栏被移除后，系统不会再自动绘制标题栏小图标。当前自绘逻辑只绘制了标题栏背景、按钮和标题文本，没有绘制窗口图标。

## 目标

在保留现有自绘标题栏、彩色 Emoji 标题文本、标题栏按钮与主题色逻辑的前提下，让标题栏在“窗口已设置图标”时显示小图标；若未设置图标，则不显示任何占位图标。

## 约束

- 保持 `设置窗口图标()` 现有接口和行为不变。
- 不关闭自定义标题栏。
- 不引入额外窗口状态字段，优先做最小改动。
- 不影响任务栏图标显示。

## 方案对比

### 方案 A：绘制时直接读取窗口当前图标（采用）

在 `DrawWindowTitleBar()` 中按顺序读取：

1. `WM_GETICON + ICON_SMALL`
2. `WM_GETICON + ICON_BIG`
3. `GCLP_HICONSM`
4. `GCLP_HICON`

若读取到图标，则在标题栏左侧绘制，并把标题文本起始位置向右偏移；若读取不到，则保持现有布局。

**优点**
- 改动最小
- 不需要修改 `WindowState`
- 与现有 `set_window_icon()` 兼容

**缺点**
- 每次重绘都需要读取一次图标句柄

### 方案 B：在 `set_window_icon()` 中缓存图标到 `WindowState`

优点是绘制时更直接，但需要新增状态字段和资源管理逻辑，不适合当前最小修复。

### 方案 C：仅依赖类图标

兼容性不如方案 A，对运行时 `WM_SETICON` 支持较弱，不采用。

## 采用方案

采用方案 A，并使用 D2D 完成图标绘制，避免与当前 D2D 标题栏绘制链混用 GDI 导致覆盖或闪烁。

## 设计细节

### 图标获取逻辑

新增一个小型辅助函数，传入 `HWND`，按优先级返回可用的 `HICON`：

1. 小图标优先
2. 其次大图标
3. 再回退到窗口类图标

### 图标绘制逻辑

在 `DrawWindowTitleBar()` 中：

- 若存在图标：
  - 以约 `16x16` 尺寸绘制在标题栏左侧
  - 垂直居中对齐
- 若不存在图标：
  - 跳过图标绘制
  - 标题文本维持原有左边距

### 标题文本布局

当前标题文本左边距固定为 `10px`。

调整为：
- 无图标：`10px`
- 有图标：`10px + 16px + 8px`

右侧仍保留标题栏按钮区域。

5. 标题文字与图标不重叠。
6. 标题栏按钮（最小化、最大化、关闭）行为不受影响。
