# 已知容易白屏的状况

## 结论

当前排查结果表明，`TabControl` 本身不是白屏根因，`AddTab页 (..., 0)` 自动创建内容窗口本身也不是白屏根因。

真正的高风险组合是：

- `TabControl` 内容窗口中放入编辑框类控件
- 同时发生窗口拖拽缩放

在这个组合下，容易出现：

- 拖拽窗口大小后内容区一直白屏
- 启动后直接白屏卡死
- 白屏后不再恢复，最终崩溃

## 已验证稳定的组合

以下组合在当前环境下已验证稳定，不白屏：

1. `TabControl` + `AddTab页 (..., 0)` 自动创建内容窗口 + 普通 `Label`
2. `TabControl` + `AddTab页 (..., 0)` 自动创建内容窗口 + 普通 `Button` + 普通 `Label`

对应验证版思路：

- 最小稳定版：每个 Tab 内容窗口里只放一个普通标签
- 按钮中间稳定版：在最小稳定版基础上恢复后退、前进、刷新、主页四个普通按钮

## 已验证高风险的组合

以下组合在当前环境下已验证高风险：

1. `TabControl` 内容窗口 + 编辑框 + 窗口拖拽缩放
2. `TabControl` 内容窗口 + 多个子控件中包含编辑框 + 窗口拖拽缩放

已出现过的问题：

- 普通编辑框版本启动后直接卡死白屏，然后崩溃
- 早期版本在拖拽窗口大小后一直白屏
- 即使补 `强制重绘TabControl`、`InvalidateRect`、`UpdateWindow`、`ShowWindow`，仍然不能稳定恢复

## 当前判断

根据分层排查结果，可以把问题收敛为：

- `TabControl` 没问题
- `TabControl` 自动创建的内容窗口没问题
- 普通 `Label` 没问题
- 普通 `Button` 没问题
- 编辑框链路是当前最可疑、最不稳定的部分

也就是说，当前最可能的根因是：

`TabControl 内容窗口 + 编辑框 + resize`

## 建议

### 稳定优先

建议在 `TabControl` 自动创建的内容窗口中，只放：

- `Label`
- `Button`

暂时不要在 `TabControl` 内容窗口中直接放：

- 地址栏编辑框
- 彩色编辑框
- 其他编辑类控件

### 如果必须要有地址栏

建议把地址栏放在主窗口外层，不要放到 `TabControl` 的内容窗口里。

也就是：

- 顶部标签栏用 `TabControl`
- 地址栏放主窗口
- 每个 Tab 内容窗口内部只承载稳定的按钮/标签/页面内容容器

## 建议的后续排查顺序

如果后面还要继续验证，请严格按这个顺序做增量排查：

1. `TabControl` 内容窗口里只放 `Label`
2. 再加普通 `Button`
3. 最后单独加一个普通 `EditBox`
4. 最后再测试彩色/特殊编辑框

不要一次性把按钮、编辑框、复杂内容一起加回去，否则无法判断到底是哪一层触发白屏。

## 本次排查的有效结论

- “最小稳定版”不白屏
- “只恢复按钮”的中间稳定版不白屏
- 一旦恢复编辑框链路，就再次进入高风险区

因此当前可以把工程经验先记为：

> `TabControl` 自动创建的内容窗口里，按钮和标签基本稳定；编辑框是已知容易引发白屏的问题点。
