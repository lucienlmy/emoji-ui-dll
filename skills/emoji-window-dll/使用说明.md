# Emoji Window DLL Skill 使用说明

## 这个 Skill 是什么

这是一个为 AI 编程助手准备的知识包，包含 `emoji_window.dll`（一个 C++ Direct2D 渲染的 Windows UI 控件库）的完整 API 文档、编码规则和代码模板。

当 AI 加载了这个 Skill 后，它能够：

- 为 **易语言** 生成正确的 DLL 调用代码（包括正确处理全角符号、UTF-8 字节集、Emoji 硬编码等易语言特有陷阱）
- 为 **Python** 生成正确的 ctypes 调用代码（包括函数签名设置、回调防 GC、两次调用模式等）
- 为 **C#** 生成正确的 P/Invoke 调用代码（包括 DllImport 声明、Marshal 内存操作、委托定义等）
- 熟悉全部 **16 种控件** 的创建、属性设置、事件回调 API
- 正确使用 **ARGB 颜色格式**、**两次调用模式**、**Element UI 配色**
- 生成包含 **布局管理器**、**主题系统**、**扩展事件** 的完整应用代码

## 在 Kiro 中使用

### 自动触发

在 Kiro 项目的 Steering 或对话中提到以下关键词时，Kiro 会自动加载本 Skill：

> emoji_window、dll、易语言、epl、direct2d、ui控件、按钮、标签、编辑框、复选框、进度条、列表框、组合框、图片框、分组框、热键、树形框、表格、选项卡、信息框、ctypes、DllImport

### 手动引用

在对话中手动引用 Skill：

```
@skill Emoji Window DLL
```

或者在 Steering 文件（`.kiro/steering/*.md`）中引用：

```markdown
#[[skill:Emoji Window DLL]]
```

## 在 Cursor 中使用

### 方法 1：作为 Rules 引用


### 方法 2：对话中手动附加

在 Cursor Chat 中用 `@` 符号附加文件：

## 在其他 AI 客户端中使用

### Claude / ChatGPT / 通用 AI

将相关文件内容作为上下文粘贴到对话中。推荐优先级：

1. **必须**：`00-overview.md`（通用规则和颜色格式）
2. **必须**：对应语言的规则文件（如 `rules/epl-rules.md`）
3. **按需**：需要用到的控件文档（如 `controls/button.md`）
4. **推荐**：对应语言的模板文件（如 `templates/epl-template.md`）

## 典型使用场景

### 场景 1：生成易语言窗口程序

> **提问**：帮我用易语言写一个登录窗口，包含用户名编辑框、密码编辑框和登录按钮

AI 会根据 Skill 中的规则自动：
- 使用 `创建Emoji窗口_字节集` 而非 `创建Emoji窗口`
- 使用全角符号 `＝` `＋`
- 正确处理 UTF-8 字节集
- 保存按钮 ID 到程序集变量
- 使用正确的回调签名

### 场景 2：生成 Python 自动化脚本

> **提问**：用 Python 写一个 emoji_window.dll 的 GUI 程序，显示一个表格并支持排序

AI 会根据 Skill 自动：
- 正确设置 ctypes 函数签名
- 使用 WINFUNCTYPE 定义回调
- 保存回调引用防止 GC
- 使用两次调用模式获取文本

### 场景 3：生成 C# 应用

> **提问**：用 C# 调用 emoji_window.dll 创建一个带有树形菜单和选项卡的应用

AI 会根据 Skill 自动：
- 生成正确的 DllImport 声明
- 使用 UnmanagedFunctionPointer 委托
- 用 static 字段防止委托被 GC
- 使用 GetText2Call 辅助方法

### 场景 4：查询特定控件 API

> **提问**：DataGridView 的虚拟模式怎么用？回调函数签名是什么？

AI 会直接从 `controls/datagridview.md` 中找到准确的 API 参数和回调签名。

## 这个 Skill 解决了什么问题

没有这个 Skill 时，AI 生成 emoji_window.dll 的调用代码经常会：

| 问题 | 没有 Skill | 有 Skill |
|------|-----------|---------|
| 易语言符号 | 用半角 `=` `+` 导致编译错误 | 自动用全角 `＝` `＋` |
| 获取文本 | 直接调用一次获取空数据 | 正确使用两次调用模式 |
| Emoji 文本 | 用 `到文本()` 导致乱码 | 全程字节集拼接 |
| 按钮 ID | 硬编码数字 1、2、3 | 保存到程序集变量 |
| 回调函数 | 参数个数错误 | 使用正确签名 |
| Python 回调 | 被 GC 回收后崩溃 | 保存到实例变量 |
| C# 委托 | 运行时异常 | static 字段 + 正确 CallingConvention |
| 颜色格式 | 用 RGB 传 ARGB 参数 | 正确使用 `(A<<24)\|(R<<16)\|(G<<8)\|B` |
