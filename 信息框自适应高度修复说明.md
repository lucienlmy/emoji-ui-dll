# 信息框自适应高度修复说明

## 🐛 问题描述

从用户提供的截图可以看出，当信息框内容较多（多行文本）时，按钮位置没有根据内容高度自适应调整，导致：
- 按钮可能覆盖在文本上
- 或者按钮位置固定，文本显示不全

## 🔍 问题根源

在 `CreateMessageBoxWindow` 函数中，文本高度测量存在以下问题：

### 1. 字体不一致
```cpp
// ❌ 错误：测量时使用 Segoe UI Emoji
g_dwrite_factory->CreateTextFormat(
    L"Segoe UI Emoji",  // 测量用字体
    ...
);

// 但实际绘制时使用 Microsoft YaHei UI
factory->CreateTextFormat(
    L"Microsoft YaHei UI",  // 绘制用字体
    ...
);
```

**问题**：不同字体的字符宽度和高度不同，导致测量结果不准确。

### 2. 行高设置不一致
```cpp
// ❌ 错误：测量时行高 21.0f
temp_format->SetLineSpacing(DWRITE_LINE_SPACING_METHOD_UNIFORM, 21.0f, 14.0f);

// 但绘制时行高 22.0f
msg_format->SetLineSpacing(DWRITE_LINE_SPACING_METHOD_UNIFORM, 22.0f, 14.0f);
```

**问题**：行高不一致导致多行文本的总高度计算错误。

### 3. 文本区域宽度不一致
```cpp
// ❌ 错误：测量时宽度 box_width - 48 (左右各 24px 边距)
g_dwrite_factory->CreateTextLayout(
    message.c_str(),
    (UINT32)message.length(),
    temp_format,
    (float)(box_width - 48),  // 文本宽度
    1000.0f,
    &text_layout
);

// 但绘制时宽度 width - 60 (左右各 30px 边距)
D2D1_RECT_F msg_rect = D2D1::RectF(30, msg_top, width - 30, msg_bottom);
```

**问题**：宽度不一致导致换行位置不同，进而影响总高度。

### 4. 布局高度计算不准确
```cpp
// ❌ 错误：布局计算过于简化
float content_height = 40 + 30 + measured_text_height + 25 + 40 + 15;
```

**问题**：没有考虑实际的布局结构（顶部装饰条、图标区域、标题、间距等）。

---

## ✅ 修复方案

### 1. 统一字体设置
```cpp
// ✅ 正确：测量和绘制使用相同字体
IDWriteTextFormat* temp_format = nullptr;
g_dwrite_factory->CreateTextFormat(
    L"Microsoft YaHei UI",  // ✅ 与 DrawMsgBox 一致
    nullptr,
    DWRITE_FONT_WEIGHT_NORMAL,
    DWRITE_FONT_STYLE_NORMAL,
    DWRITE_FONT_STRETCH_NORMAL,
    14.0f,
    L"zh-CN",
    &temp_format
);
```

### 2. 统一行高设置
```cpp
// ✅ 正确：测量和绘制使用相同行高
temp_format->SetLineSpacing(DWRITE_LINE_SPACING_METHOD_UNIFORM, 22.0f, 14.0f);  // ✅ 1.57x line height
```

### 3. 统一文本区域宽度
```cpp
// ✅ 正确：测量时使用与绘制相同的宽度
g_dwrite_factory->CreateTextLayout(
    message.c_str(),
    (UINT32)message.length(),
    temp_format,
    (float)(box_width - 60),  // ✅ 左右各 30px 边距，与 DrawMsgBox 一致
    2000.0f,  // 增加最大高度以支持长消息
    &text_layout
);
```

### 4. 精确的布局高度计算
```cpp
// ✅ 正确：根据实际布局结构计算
// Layout breakdown:
// - Top bar: 3px (蓝色装饰条)
// - Icon area: 70px (20px top + 50px icon)
// - Title: 30px
// - Message top spacing: 10px
// - Message: variable (measured_text_height)
// - Message bottom spacing: 20px
// - Button: 40px
// - Bottom padding: 15px
float content_height = 3 + 70 + 30 + 10 + measured_text_height + 20 + 40 + 15;
int min_height = (type == MSGBOX_OKCANCEL) ? 240 : 220;  // 最小高度
int box_height = (int)max(content_height, (float)min_height);

// 按钮 Y 坐标根据实际窗口高度计算
int btn_y = box_height - 55;  // 40px button + 15px bottom padding
```

### 场景 1：短消息（单行）
```
消息：您确定要发射火箭吗？ 🚀
预期：窗口高度 = 最小高度 (240px for OKCANCEL, 220px for OK)
```

```
消息：您确定要发射火箭吗？ 🚀
     这是一个不可逆的操作！
预期：窗口高度 = 188 + (2-3行文本高度) ≈ 250-280px
```

### 场景 3：长消息（多行）
```
消息：您确定要发射火箭吗？ 🚀
     这是一个不可逆的操作！
     发射后将无法撤回。
     请再次确认您的决定。
预期：窗口高度 = 188 + (4行文本高度) ≈ 300-350px
按钮位置：始终在窗口底部 55px 处
```
