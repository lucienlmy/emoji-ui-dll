# Tab切换问题 - 最终解决方案总结

## 问题回顾

**原始问题**：
1. 切换到Tab2时，Tab1的按钮仍然显示
2. Tab2中的编辑框不显示

**尝试的方案**：
- ❌ 在易语言层面调用 `ShowWindow` 手动隐藏Tab1窗口 → 无效

## 根本原因

问题出在DLL的绘制逻辑：

### 原因1：Direct2D绘制不检查窗口可见性

按钮是通过Direct2D在 `WM_PAINT` 消息中绘制的。即使窗口被 `ShowWindow(SW_HIDE)` 隐藏，`WM_PAINT` 消息仍然会被处理，导致按钮被绘制出来。

**原代码**（emoji_window.cpp 第163行）：
```cpp
case WM_PAINT: {
    if (state && state->render_target) {
        PAINTSTRUCT ps;
        BeginPaint(hwnd, &ps);

        state->render_target->BeginDraw();
        state->render_target->Clear(D2D1::ColorF(0xF5F5FA, 1.0f));

        // 无论窗口是否可见，都绘制按钮！
        for (const auto& button : state->buttons) {
            DrawButton(state->render_target, state->dwrite_factory, button);
        }

        state->render_target->EndDraw();
        EndPaint(hwnd, &ps);
    }
    return 0;
}
```

### 原因2：UpdateTabLayout使用的隐藏方法不够可靠

原代码使用 `SetWindowPos(..., SWP_HIDEWINDOW)` 隐藏窗口，但这个方法可能存在时序问题，导致窗口状态更新不及时。

## 解决方案

### 修改1：在WM_PAINT中添加可见性检查

```cpp
case WM_PAINT: {
    if (state && state->render_target) {
        PAINTSTRUCT ps;
        BeginPaint(hwnd, &ps);

        // ✅ 只有窗口可见时才绘制
        if (IsWindowVisible(hwnd)) {
            state->render_target->BeginDraw();
            state->render_target->Clear(D2D1::ColorF(0xF5F5FA, 1.0f));

            for (const auto& button : state->buttons) {
                DrawButton(state->render_target, state->dwrite_factory, button);
            }

            state->render_target->EndDraw();
        }

        EndPaint(hwnd, &ps);
    }
    return 0;
}
```

**效果**：隐藏的窗口不会绘制按钮，彻底解决按钮重叠问题。

### 修改2：改进UpdateTabLayout函数

```cpp
void UpdateTabLayout(TabControlState* state) {
    // ... 获取显示区域代码 ...

    // ✅ 第一步：使用ShowWindow隐藏非当前Tab
    for (size_t i = 0; i < state->pages.size(); i++) {
        if ((int)i != state->currentIndex) {
            TabPageInfo& page = state->pages[i];
            if (page.hContentWindow && IsWindow(page.hContentWindow)) {
                ShowWindow(page.hContentWindow, SW_HIDE);  // 更可靠
                page.visible = false;
                InvalidateRect(page.hContentWindow, nullptr, TRUE);  // 强制刷新
            }
        }
    }

    // ✅ 第二步：分离SetWindowPos和ShowWindow调用
    if (state->currentIndex >= 0 && state->currentIndex < (int)state->pages.size()) {
        TabPageInfo& page = state->pages[state->currentIndex];
        if (page.hContentWindow && IsWindow(page.hContentWindow)) {
            // 先设置位置和大小
            SetWindowPos(page.hContentWindow, HWND_TOP, 
                        rcTab.left, rcTab.top, contentWidth, contentHeight,
                        SWP_NOACTIVATE);
            
            // 再显示窗口
            ShowWindow(page.hContentWindow, SW_SHOW);
            page.visible = true;

            // 强制刷新
            InvalidateRect(page.hContentWindow, nullptr, TRUE);
            UpdateWindow(page.hContentWindow);
        }
    }
}
```

**效果**：更可靠地控制Tab窗口的显示/隐藏，确保编辑框正常显示。

## 技术要点

### 为什么易语言层面无法解决？

1. **Direct2D绘制机制**：
   - 按钮不是真正的子窗口控件
   - 是通过Direct2D在父窗口上绘制的图形
   - `ShowWindow` 只能控制窗口可见性，不能阻止Direct2D绘制

2. **WM_PAINT消息处理**：
   - 即使窗口隐藏，`WM_PAINT` 仍可能被调用
   - 必须在绘制代码中检查窗口可见性

### 为什么这个修改有效？

1. **IsWindowVisible检查**：
   - 在绘制前检查窗口是否可见
   - 隐藏的窗口直接跳过绘制逻辑
   - 从根源上阻止了按钮的绘制

2. **ShowWindow + InvalidateRect**：
   - `ShowWindow` 更新窗口可见性标志
   - `InvalidateRect` 触发 `WM_PAINT` 消息
   - `WM_PAINT` 中的 `IsWindowVisible` 检查生效
   - 形成完整的显示/隐藏循环
