# 终极解决方案 - 易语言层面强制控制

## 问题确认

DLL修改后：
- ✅ 窗口可见性正确（Tab1可见: 假，Tab2可见: 真）
- ❌ 但Tab1按钮仍然可见
- ❌ 编辑框仍然不可见

这说明问题在于**窗口的Z顺序和绘制缓存**。

## 终极解决方案

在易语言的Tab切换回调中，添加更激进的控制：

```易语言
.子程序 Tab切换回调, , 公开
.参数 TabControl, 整数型
.参数 当前索引, 整数型
.局部变量 Tab1窗口, 整数型
.局部变量 Tab2窗口, 整数型
.局部变量 Tab3窗口, 整数型

调试输出 ("========== Tab切换到索引: " ＋ 到文本 (当前索引) ＋ " ==========")

Tab1窗口 ＝ 获取Tab内容窗口 (TabControl, 0)
Tab2窗口 ＝ 获取Tab内容窗口 (TabControl, 1)
Tab3窗口 ＝ 获取Tab内容窗口 (TabControl, 2)

.判断开始 (当前索引 ＝ 0)
    ' ========== 显示Tab1 ==========
    ' 1. 先隐藏其他Tab并移到底部
    SetWindowPos (Tab2窗口, 1, 0, 0, 0, 0, 131)  ' HWND_BOTTOM=1, SWP_NOMOVE|SWP_NOSIZE|SWP_HIDEWINDOW=131
    SetWindowPos (Tab3窗口, 1, 0, 0, 0, 0, 131)
    
    ' 2. 显示Tab1并移到顶部
    SetWindowPos (Tab1窗口, 0, 0, 0, 0, 0, 67)  ' HWND_TOP=0, SWP_NOMOVE|SWP_NOSIZE|SWP_SHOWWINDOW=67
    
    ' 3. 强制重绘
    RedrawWindow (Tab1窗口, 0, 0, 1)  ' RDW_INVALIDATE=1
    RedrawWindow (Tab2窗口, 0, 0, 1)
    
.判断 (当前索引 ＝ 1)
    ' ========== 显示Tab2 ==========
    ' 1. 先隐藏其他Tab并移到底部
    SetWindowPos (Tab1窗口, 1, 0, 0, 0, 0, 131)  ' HWND_BOTTOM=1, SWP_NOMOVE|SWP_NOSIZE|SWP_HIDEWINDOW=131
    SetWindowPos (Tab3窗口, 1, 0, 0, 0, 0, 131)
    
    ' 2. 显示Tab2并移到顶部
    SetWindowPos (Tab2窗口, 0, 0, 0, 0, 0, 67)  ' HWND_TOP=0, SWP_NOMOVE|SWP_NOSIZE|SWP_SHOWWINDOW=67
    
    ' 3. 强制重绘Tab2及其所有子窗口
    RedrawWindow (Tab2窗口, 0, 0, 5)  ' RDW_INVALIDATE|RDW_ALLCHILDREN=5
    
    ' 4. 确保编辑框在顶部并可见
    .如果真 (编辑框句柄 ≠ 0)
        SetWindowPos (编辑框句柄, 0, 0, 0, 0, 0, 67)  ' HWND_TOP, SHOWWINDOW
        RedrawWindow (编辑框句柄, 0, 0, 1)
        调试输出 ("强制显示编辑框")
    .如果真结束
    
    ' 5. 再次确认Tab1被隐藏
    ShowWindow (Tab1窗口, 0)  ' SW_HIDE
    RedrawWindow (Tab1窗口, 0, 0, 1)
    
.判断 (当前索引 ＝ 2)
    ' ========== 显示Tab3 ==========
    SetWindowPos (Tab1窗口, 1, 0, 0, 0, 0, 131)
    SetWindowPos (Tab2窗口, 1, 0, 0, 0, 0, 131)
    SetWindowPos (Tab3窗口, 0, 0, 0, 0, 0, 67)
    RedrawWindow (Tab3窗口, 0, 0, 1)
.判断结束

' 等待窗口更新
处理事件 ()

调试输出 ("========================================")
```

## 需要添加的DLL命令

```易语言
.DLL命令 SetWindowPos, 逻辑型, "user32.dll", "SetWindowPos"
    .参数 窗口句柄, 整数型
    .参数 插入位置, 整数型
    .参数 X, 整数型
    .参数 Y, 整数型
    .参数 宽度, 整数型
    .参数 高度, 整数型
    .参数 标志, 整数型

.DLL命令 RedrawWindow, 逻辑型, "user32.dll", "RedrawWindow"
    .参数 窗口句柄, 整数型
    .参数 矩形指针, 整数型
    .参数 区域句柄, 整数型
    .参数 标志, 整数型
```

## 常量说明

```易语言
' SetWindowPos 的 hWndInsertAfter 参数
' HWND_TOP = 0 (置于顶部)
' HWND_BOTTOM = 1 (置于底部)

' SetWindowPos 的 uFlags 参数
' SWP_NOMOVE = 2 (不改变位置)
' SWP_NOSIZE = 1 (不改变大小)
' SWP_SHOWWINDOW = 64 (显示窗口)
' SWP_HIDEWINDOW = 128 (隐藏窗口)
' 组合：67 = 1+2+64 (不改变位置和大小，显示窗口)
' 组合：131 = 1+2+128 (不改变位置和大小，隐藏窗口)

' RedrawWindow 的 flags 参数
' RDW_INVALIDATE = 1 (使窗口无效，触发重绘)
' RDW_ALLCHILDREN = 4 (包括所有子窗口)
' 组合：5 = 1+4 (重绘窗口及所有子窗口)
```

## 为什么这个方案应该有效

1. **SetWindowPos 同时控制Z顺序和可见性**
   - 一次性将窗口移到底部并隐藏
   - 避免中间状态

2. **RedrawWindow 强制重绘**
   - 清除绘制缓存
   - 包括所有子窗口

3. **多重保障**
   - 先隐藏其他Tab
   - 再显示当前Tab
   - 最后强制重绘

## 如果还不行

如果这个方案仍然无效，那么问题可能是：

### 1. Direct2D的渲染目标共享

所有Tab内容窗口可能共享同一个Direct2D渲染目标，导致绘制内容混乱。

**解决方案**：修改DLL，为每个Tab创建独立的渲染目标。

### 2. 窗口类的问题

Tab内容窗口使用的是 `EmojiWindowClass`，这个类可能有特殊的绘制逻辑。

**解决方案**：为Tab内容窗口创建专用的窗口类。

### 3. 父窗口的WS_CLIPCHILDREN标志

主窗口的 `WS_CLIPCHILDREN` 标志可能影响子窗口的绘制。

**解决方案**：修改窗口样式。
