## 修改内容

已成功修改 `emoji_window.cpp`，解决Tab切换问题：

### 修改1：WindowProc的WM_PAINT处理

**位置**：第163行左右

**修改内容**：添加 `IsWindowVisible(hwnd)` 检查

```cpp
case WM_PAINT: {
    if (state && state->render_target) {
        PAINTSTRUCT ps;
        BeginPaint(hwnd, &ps);

        // ========== 添加可见性检查，防止隐藏窗口仍然绘制 ==========
        if (IsWindowVisible(hwnd)) {
            state->render_target->BeginDraw();
            state->render_target->Clear(D2D1::ColorF(0xF5F5FA, 1.0f));

            for (const auto& button : state->buttons) {
                DrawButton(state->render_target, state->dwrite_factory, button);
            }

            state->render_target->EndDraw();
        }
        // ========================================

        EndPaint(hwnd, &ps);
    } else {
        return DefWindowProc(hwnd, msg, wparam, lparam);
    }
    return 0;
}
```

**作用**：只有当窗口可见时才绘制按钮，隐藏的窗口不会绘制内容。

### 修改2：UpdateTabLayout函数

**位置**：第950行左右

**修改内容**：
1. 使用 `ShowWindow(SW_HIDE)` 而不是 `SetWindowPos(..., SWP_HIDEWINDOW)`
2. 添加 `InvalidateRect(..., TRUE)` 强制触发重绘
3. 分离 `SetWindowPos` 和 `ShowWindow` 调用

**作用**：更可靠地隐藏非当前Tab的窗口，确保窗口状态正确更新。

## 如果编译失败

### 常见错误1：找不到头文件

**错误信息**：`fatal error C1083: Cannot open include file: 'd2d1.h'`

**解决方案**：
- 确保安装了Windows SDK
- 使用"x86 Native Tools Command Prompt"而不是普通命令提示符

### 常见错误2：链接错误

**错误信息**：`error LNK2019: unresolved external symbol`

**解决方案**：
- 确保链接了所有必需的库：`d2d1.lib dwrite.lib comctl32.lib uxtheme.lib`
- 检查函数调用约定（`__stdcall`）

### 常见错误3：平台不匹配

**错误信息**：易语言运行时提示"无法加载DLL"

**解决方案**：
- 确保编译为32位（x86），不是64位（x64）
- 易语言是32位程序，只能加载32位DLL
