# 编辑框和标签闪烁修复说明

## 问题描述

在 Tab3 中创建的编辑框和标签组件，鼠标移动时会出现闪烁现象。

## 问题原因

### 根本原因 1：GDI 资源泄漏和重复创建

在 `EditBoxSubclassProc` 和 `LabelSubclassProc` 的消息处理中，每次收到 `WM_CTLCOLOREDIT`、`WM_CTLCOLORSTATIC` 或 `WM_PAINT` 消息时，都会调用 `CreateSolidBrush()` 创建新的画刷，但从不删除旧的画刷，导致：

1. **GDI 资源泄漏**：每次鼠标移动都会创建新的画刷对象，但从不释放
2. **重绘闪烁**：频繁创建和使用新画刷导致重绘不流畅，产生闪烁效果
3. **性能下降**：随着时间推移，GDI 资源耗尽，系统性能下降

### 根本原因 2：父窗口缺少 WS_CLIPCHILDREN 样式

Tab 内容窗口在创建时使用了 `WS_CLIPSIBLINGS` 但缺少 `WS_CLIPCHILDREN` 样式，导致：

1. **父窗口重绘时覆盖子控件**：父窗口重绘时会绘制到子控件区域
2. **子控件需要频繁重绘**：每次父窗口重绘都会触发子控件重绘
3. **视觉闪烁**：父窗口和子控件的重绘不同步，产生闪烁

### 根本原因 3：父窗口未处理 WM_CTLCOLOREDIT 消息

`WM_CTLCOLOREDIT` 和 `WM_CTLCOLORSTATIC` 消息是发送到**父窗口**的，而不是子控件本身。如果父窗口不处理这些消息，系统会使用默认处理，可能导致：

1. **颜色设置不生效**：子类化处理中的颜色设置可能被覆盖
2. **重绘不一致**：父窗口和子控件的绘制不协调
3. **闪烁加剧**：多次重绘导致闪烁更明显

### 错误代码示例

```cpp
// ❌ 错误的实现 - EditBoxSubclassProc
case WM_CTLCOLOREDIT:
case WM_CTLCOLORSTATIC: {
    HDC hdc = (HDC)wparam;
    SetTextColor(hdc, ...);
    SetBkColor(hdc, ...);
    // ❌ 每次都创建新画刷，但从不删除！
    return (LRESULT)CreateSolidBrush(RGB(...));
}

// ❌ 错误的实现 - LabelSubclassProc
case WM_PAINT: {
    // ❌ 每次绘制都创建新画刷
    HBRUSH bgBrush = CreateSolidBrush(RGB(...));
    FillRect(hdc, &rect, bgBrush);
    DeleteObject(bgBrush);  // 虽然删除了，但频繁创建仍会导致闪烁
    ...
}
```

## 修复方案

### 核心思路

1. **画刷复用**：创建一次画刷，保存在状态结构中，多次使用
2. **添加 WS_CLIPCHILDREN 样式**：防止父窗口重绘时覆盖子控件
3. **父窗口处理颜色消息**：在父窗口的 WindowProc 中处理 `WM_CTLCOLOREDIT` 和 `WM_CTLCOLORSTATIC` 消息

#### 2. 修改 EditBoxSubclassProc（emoji_window.cpp）

```cpp
LRESULT CALLBACK EditBoxSubclassProc(HWND hwnd, UINT msg, WPARAM wparam, LPARAM lparam, UINT_PTR uIdSubclass, DWORD_PTR dwRefData) {
    EditBoxState* state = (EditBoxState*)dwRefData;
    
    switch (msg) {
        case WM_CTLCOLOREDIT:
        case WM_CTLCOLORSTATIC: {
            HDC hdc = (HDC)wparam;
            SetTextColor(hdc, RGB(
                (state->fg_color >> 16) & 0xFF,
                (state->fg_color >> 8) & 0xFF,
                state->fg_color & 0xFF
            ));
            SetBkColor(hdc, RGB(
                (state->bg_color >> 16) & 0xFF,
                (state->bg_color >> 8) & 0xFF,
                state->bg_color & 0xFF
            ));
            // ✅ 返回保存的画刷，而不是每次创建新的
            return (LRESULT)state->bg_brush;
        }
        case WM_NCDESTROY: {
            // ✅ 清理画刷资源
            if (state->bg_brush) {
                DeleteObject(state->bg_brush);
                state->bg_brush = nullptr;
            }
            RemoveWindowSubclass(hwnd, EditBoxSubclassProc, uIdSubclass);
            break;
        }
    }
    
    return DefSubclassProc(hwnd, msg, wparam, lparam);
}
```

#### 6. 修改 SetEditBoxColor 和 SetLabelColor（emoji_window.cpp）

```cpp
__declspec(dllexport) void __stdcall SetEditBoxColor(
    HWND hEdit,
    UINT32 fg_color,
    UINT32 bg_color
) {
    auto it = g_editboxes.find(hEdit);
    if (it == g_editboxes.end()) return;
    
    EditBoxState* state = it->second;
    state->fg_color = fg_color;
    state->bg_color = bg_color;
    
    // ✅ 重新创建背景画刷
    if (state->bg_brush) {
        DeleteObject(state->bg_brush);
    }
    state->bg_brush = CreateSolidBrush(RGB(
        (bg_color >> 16) & 0xFF,
        (bg_color >> 8) & 0xFF,
        bg_color & 0xFF
    ));
    
    InvalidateRect(hEdit, NULL, TRUE);
}
```

#### 8. 在 WindowProc 中添加颜色消息处理（emoji_window.cpp）

```cpp
LRESULT CALLBACK WindowProc(HWND hwnd, UINT msg, WPARAM wparam, LPARAM lparam) {
    // ... 其他代码 ...
    
    switch (msg) {
    // ... 其他消息处理 ...
    
    case WM_CTLCOLOREDIT:
    case WM_CTLCOLORSTATIC: {
        // ✅ 处理编辑框和标签的颜色消息
        // 这些消息会发送到父窗口，我们需要查找对应的子控件状态
        HWND hChild = (HWND)lparam;
        
        // 先查找编辑框
        auto edit_it = g_editboxes.find(hChild);
        if (edit_it != g_editboxes.end()) {
            EditBoxState* edit_state = edit_it->second;
            HDC hdc = (HDC)wparam;
            SetTextColor(hdc, RGB(
                (edit_state->fg_color >> 16) & 0xFF,
                (edit_state->fg_color >> 8) & 0xFF,
                edit_state->fg_color & 0xFF
            ));
            SetBkColor(hdc, RGB(
                (edit_state->bg_color >> 16) & 0xFF,
                (edit_state->bg_color >> 8) & 0xFF,
                edit_state->bg_color & 0xFF
            ));
            return (LRESULT)edit_state->bg_brush;
        }
        
        // 再查找标签
        auto label_it = g_labels.find(hChild);
        if (label_it != g_labels.end()) {
            LabelState* label_state = label_it->second;
            HDC hdc = (HDC)wparam;
            SetTextColor(hdc, RGB(
                (label_state->fg_color >> 16) & 0xFF,
                (label_state->fg_color >> 8) & 0xFF,
                label_state->fg_color & 0xFF
            ));
            SetBkColor(hdc, RGB(
                (label_state->bg_color >> 16) & 0xFF,
                (label_state->bg_color >> 8) & 0xFF,
                label_state->bg_color & 0xFF
            ));
            return (LRESULT)label_state->bg_brush;
        }
        
        break;
    }
    
    // ... 其他消息处理 ...
    }
    
    return DefWindowProc(hwnd, msg, wparam, lparam);
}
```

## 技术要点

### 1. GDI 对象生命周期管理

GDI 对象（如画刷、字体、画笔等）是系统资源，必须正确管理：

- **创建**：使用 `CreateSolidBrush()`、`CreateFont()` 等函数
- **使用**：在绘制时使用创建的对象
- **释放**：使用 `DeleteObject()` 释放不再使用的对象

### 2. 避免频繁创建 GDI 对象

频繁创建和销毁 GDI 对象会导致：
- 性能下降
- 内存碎片
- 视觉闪烁

**最佳实践**：
- 创建一次，多次使用
- 在对象生命周期结束时统一释放
- 只在必要时（如颜色改变）重新创建

### 3. 子类化消息处理

Windows 子类化允许拦截和处理控件的消息：

```cpp
// 设置子类化
SetWindowSubclass(hwnd, SubclassProc, 0, (DWORD_PTR)state);

// 子类化处理函数
LRESULT CALLBACK SubclassProc(HWND hwnd, UINT msg, WPARAM wparam, LPARAM lparam, 
                              UINT_PTR uIdSubclass, DWORD_PTR dwRefData) {
    // 处理消息
    switch (msg) {
        case WM_NCDESTROY:
            // ⚠️ 重要：在控件销毁时移除子类化
            RemoveWindowSubclass(hwnd, SubclassProc, uIdSubclass);
            break;
    }
    
    // 调用默认处理
    return DefSubclassProc(hwnd, msg, wparam, lparam);
}
```

### 4. WM_CTLCOLOREDIT 和 WM_CTLCOLORSTATIC 消息

这些消息用于自定义编辑框和静态控件的颜色：

- **WM_CTLCOLOREDIT**：编辑框颜色控制
- **WM_CTLCOLORSTATIC**：静态控件（标签）颜色控制

**返回值**：必须返回一个画刷句柄（HBRUSH），系统会使用这个画刷绘制控件背景

**⚠️ 注意**：返回的画刷不应该在消息处理中删除，因为系统可能会多次使用它

## 总结

这次修复的核心是**资源复用**和**生命周期管理**：

1. **问题**：每次消息处理都创建新的 GDI 对象
2. **方案**：创建一次，保存在状态结构中，多次使用
3. **清理**：在控件销毁时统一释放资源

这种模式适用于所有需要频繁使用的 GDI 对象，是 Windows GDI 编程的最佳实践。
