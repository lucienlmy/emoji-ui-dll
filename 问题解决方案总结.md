# TabControl 问题分析和解决方案总结

## 问题描述

你遇到了两个问题：

1. **Tab1的按钮显示在Tab2中** - 切换到Tab2时，Tab1的按钮仍然可见
2. **编辑框没有显示** - 在Tab2中创建的编辑框不可见

## 根本原因

### 问题分析

通过分析DLL源码（`emoji_window.cpp`），发现：

1. **Tab内容窗口的创建**：
   - 每个Tab都有一个独立的内容窗口（使用 `EmojiWindowClass`）
   - 所有Tab内容窗口都是主窗口的子窗口
   - 它们的位置是重叠的（都在TabControl的显示区域内）

2. **UpdateTabLayout 函数的逻辑**：
   ```cpp
   // 第一步：隐藏非当前Tab
   SetWindowPos(page.hContentWindow, HWND_BOTTOM, 0, 0, 0, 0, 
                SWP_NOMOVE | SWP_NOSIZE | SWP_HIDEWINDOW);
   
   // 第二步：显示当前Tab
   SetWindowPos(page.hContentWindow, HWND_TOP, rcTab.left, rcTab.top, 
                contentWidth, contentHeight, SWP_SHOWWINDOW);
   ```

3. **问题所在**：
   - `SetWindowPos` 的 `SWP_HIDEWINDOW` 标志可能没有立即生效
   - 或者窗口的绘制（WM_PAINT）在隐藏之前就已经发生
   - Direct2D绘制的按钮可能不受窗口可见性影响

### 为什么编辑框不显示

编辑框是Tab2内容窗口的子窗口：
- 如果Tab2内容窗口被Tab1内容窗口遮挡，编辑框也会被遮挡
- 或者Tab2内容窗口虽然"显示"了，但Z顺序不正确

## 解决方案

### 临时解决方案（立即可用）

在易语言代码的 `Tab切换回调` 函数中手动控制窗口显示/隐藏。

**优点**：
- 不需要重新编译DLL
- 立即可以测试
- 如果有效，证明了问题所在

**实施步骤**：

1. **添加Windows API声明**（见 `Tab切换修复补丁.txt`）
2. **修改Tab切换回调函数**（见 `Tab切换修复补丁.txt`）
3. **编译运行测试**

### 永久解决方案（需要修改DLL）

修改 `emoji_window.cpp` 中的 `UpdateTabLayout` 函数：

```cpp
void UpdateTabLayout(TabControlState* state) {
    if (!state || !state->hTabControl) return;

    // 获取显示区域
    RECT rcTab;
    GetClientRect(state->hTabControl, &rcTab);
    TabCtrl_AdjustRect(state->hTabControl, FALSE, &rcTab);
    
    // 转换坐标...
    
    // ========== 关键修复：使用ShowWindow而不是SetWindowPos ==========
    
    // 第一步：先隐藏所有非当前Tab
    for (size_t i = 0; i < state->pages.size(); i++) {
        if ((int)i != state->currentIndex) {
            TabPageInfo& page = state->pages[i];
            if (page.hContentWindow && IsWindow(page.hContentWindow)) {
                ShowWindow(page.hContentWindow, SW_HIDE);  // 使用ShowWindow
                page.visible = false;
            }
        }
    }
    
    // 第二步：显示并更新当前Tab
    if (state->currentIndex >= 0 && state->currentIndex < (int)state->pages.size()) {
        TabPageInfo& page = state->pages[state->currentIndex];
        if (page.hContentWindow && IsWindow(page.hContentWindow)) {
            // 先设置位置和大小
            SetWindowPos(page.hContentWindow, HWND_TOP, 
                        rcTab.left, rcTab.top, contentWidth, contentHeight,
                        SWP_NOACTIVATE);
            
            // 再显示窗口
            ShowWindow(page.hContentWindow, SW_SHOW);
            page.visible = true;
            
            // 更新渲染目标...
            
            // 强制刷新
            InvalidateRect(page.hContentWindow, nullptr, TRUE);
            UpdateWindow(page.hContentWindow);
        }
    }
}
```

**关键改进**：
1. 使用 `ShowWindow(SW_HIDE)` 而不是 `SetWindowPos(..., SWP_HIDEWINDOW)`
2. 分两步：先隐藏所有，再显示当前
3. 添加 `UpdateWindow` 强制立即刷新

### 另一个可能的修复（WindowProc）

在 `WindowProc` 的 `WM_PAINT` 处理中添加可见性检查：

```cpp
case WM_PAINT: {
    if (state) {
        // 检查窗口是否可见
        if (!IsWindowVisible(hwnd)) {
            // 窗口不可见，不绘制
            PAINTSTRUCT ps;
            BeginPaint(hwnd, &ps);
            EndPaint(hwnd, &ps);
            return 0;
        }
        
        // 正常绘制...
    }
    return 0;
}
```
